Determining Expected Number Of Customers Affected By Public Safety Power Shutoffs In The Next Decade

Report
Skills Used: Python, Geopandas, Networkx, Folium

The project focused on developing a comprehensive methodology to estimate the expected number of customers affected by Public Safety Power Shutoff (PSPS) events over a 10-year period for San Diego Gas & Electric (SDG&E). The goal was to quantify PSPS impact across diverse power distribution networks at span, segment, and circuit levels. The evaluation metrics used included PSPS probabilities, annual shutoff rates, and expected customer impact. The project began with extensive data analysis and merging of multiple datasets, including weather station data, wind speed records, vegetation risk indices, and conductor span information. Geospatial analysis techniques were implemented to join datasets and create visualizations of PSPS risk distribution across the service territory. A graph network object was built to represent the electrical grid and perform network analysis for tracing upstream connections. PSPS probabilities and expected shutoff rates were calculated at various levels using probabilistic methods and expected value approaches. The final analysis provided insights into the spatial distribution of PSPS risk and its implications for grid management and customer impact[1].

Skills Used

  • Data Analysis: Conducted exploratory data analysis on weather station data, wind speed records, and alert threshold information to understand PSPS risk factors.
  • Feature Engineering: Created new features such as PSPS probabilities and expected shutoff rates by combining multiple data sources and applying probabilistic calculations.
  • Graph Theory(Network x): Implemented graph-based algorithms for network analysis to trace upstream connections in the electrical grid.
  • Geospatial Analysis: Performed geospatial joins and created visualizations of PSPS risk distribution across the service territory using GIS techniques.
  • Statistical Analysis: Applied probabilistic methods, including union probability calculations, to determine PSPS probabilities and expected shutoff rates.
  • Data Visualization: Created histograms, maps, and other visual representations to illustrate PSPS risk patterns and customer impact distributions.
  • Big Data Processing: Merged and analyzed large datasets from various sources, including weather stations, vegetation risk indices, and conductor span information.
  • Programming: Utilized Python and relevant libraries (e.g., pandas, geopandas, networkx) for data manipulation, analysis, and visualization.
  • Model Evaluation: Assessed the impact of PSPS events at different granularities (span, segment, circuit) and analyzed the distribution of customer impacts.
  • Conclusion Drawing: Interpreted results to identify high-risk areas, patterns in customer impact, and implications for grid management and risk mitigation strategies.

Recipe-Ratings-Prediction-Model

Web Application
Skills Used: Pandas, Python, ML(Decision tree, Random Forest classifier)

The project focused on building a Recipe Ratings Prediction Model. The goal was to predict the ratings of recipes by extracting features from review data. The evaluation metric used was RMSE (Root Mean Square Error), which measures the difference between predicted and actual ratings. The project started with a baseline model using nutrition-related features and then evolved to include additional quantitative features like minutes, n_steps, calories, n_ingredients, reviews, and description. Hyperparameters were optimized using techniques like Polynomial Features and Grid Search CV. Different models were evaluated, including linear regression, decision trees, and random forest classifiers. The final model selected was the Random Forest Classifier due to its high R^2 and precision scores.

Skills Used

  • Data Analysis: Conducted exploratory data analysis to understand the dataset and identify relevant features for prediction.
  • Feature Engineering: Extracted quantitative features from the review data, such as minutes, n_steps, calories, n_ingredients, reviews, and description.
  • Machine Learning: Built and evaluated regression models, including linear regression, decision trees, and random forest classifiers.
  • Hyperparameter Tuning: Utilized techniques like Polynomial Features and Grid Search CV to optimize hyperparameters for the models.
  • Statistical Analysis: Performed a fairness analysis using a permutation test to compare the precision of the model for recipes with different cooking times.
  • Model Evaluation: Used RMSE, R^2, and precision scores to assess and compare the performance of different models.
  • Conclusion Drawing: Analyzed the results of the model evaluation and made conclusions based on observed differences and statistical tests.
